HTML5 API History

Allgemeines

  • Standard zum Manipulieren der Browser-History
  • Verwenden von JavaScript
  • In vorherigen Standards bereits vorhanden
    • Navigieren durch die Browser-History
  • Neu in diesem Standard
    • Einträge in die Browser-History hinzufügen
    • Die URL ändern, ohne die Seite neu laden zu müssen
    • Auslösen eines "Events" beim drücken des "Zurück"-Buttons des Browsers

Vorteile

Beispiel

Vorteile

  • Es können nur einzelne Bereiche einer Seite neu geladen werden
  • Einsparen von Ressourcen
  • Dennoch lässt sich die URL so verändern, dass sie auf eine Seite mit dem aktuellen Stand zeigt

Anwendung

  • PushState
    •                 history.pushState(stateObj, title, URL);
  • ReplaceState
    •                 history.replaceState(stateObj, title, URL);
  • In der Browser-History vor und zürück navigieren
    •                 window.history.back();
                window.history.forward();
                window.history.go(-/+ x);
